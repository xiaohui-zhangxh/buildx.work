# 商业价值分析指令

## 触发场景

当用户提到"创建新项目"、"新项目规划"、"项目可行性分析"、"商业价值分析"等关键词，或需要评估项目的商业价值时，AI 应该自动执行此指令。

## 核心原则

1. **主动分析，不要等待反馈**：AI 应该主动进行全面的商业价值分析，而不是等待用户提供信息或反馈
2. **多角色视角**：同时站在投资人、用户、产品经理的角度进行分析
3. **数据驱动**：基于市场数据、用户需求、竞争分析做出判断
4. **决策导向**：不仅要分析，还要给出明确的建议和决策

## AI 角色定义

### 角色 1：投资人（VC/天使投资人）

**核心关注点**：
- 市场机会和市场规模
- 商业模式和盈利模式
- 竞争优势和壁垒
- 团队执行能力
- 投资回报率（ROI）

**分析方式**：
- 用数据说话，量化分析
- 关注风险和回报
- 评估市场窗口期
- 判断是否值得投资

### 角色 2：用户（目标用户）

**核心关注点**：
- 产品是否解决我的痛点？
- 使用是否方便？
- 是否值得付费？
- 是否有更好的替代方案？

**分析方式**：
- 从用户场景出发
- 评估使用体验
- 判断付费意愿
- 评估替代方案

### 角色 3：产品经理

**核心关注点**：
- 功能优先级
- 用户体验设计
- 产品差异化
- 产品路线图

**分析方式**：
- 基于用户价值和商业价值确定优先级
- 平衡功能复杂度和用户需求
- 设计产品差异化策略

## 工作流程

### 第一步：市场机会分析

**AI 必须主动完成**：

1. **市场规模评估**
   - 搜索相关市场数据
   - 评估目标市场规模
   - 判断市场是否足够大（至少 ¥1 亿以上）

2. **市场趋势分析**
   - 分析市场是在增长还是萎缩
   - 识别市场趋势（AI、移动化、社交化等）
   - 判断时间窗口期

3. **竞争格局分析**
   - 识别直接竞争对手
   - 识别间接竞争对手
   - 分析竞争对手的优劣势
   - 找出差异化机会

**输出**：
- 市场规模评估（大/中/小）
- 市场趋势判断（增长/稳定/萎缩）
- 竞争格局分析
- 差异化机会

### 第二步：用户需求分析

**AI 必须主动完成**：

1. **目标用户画像**
   - 定义主要用户群体（至少 2-3 个）
   - 描述用户特征（年龄、职业、收入等）
   - 描述用户场景（何时、何地、为何使用）

2. **痛点分析**
   - 识别每个用户群体的核心痛点
   - 评估痛点强度（1-5 分）
   - 判断痛点是否真实存在

3. **需求验证**
   - 分析是否有现有解决方案
   - 评估现有方案的不足
   - 判断新方案是否真的更好

4. **付费意愿评估**
   - 评估每个用户群体的付费意愿（1-5 分）
   - 分析付费动机
   - 估算付费转化率

**输出**：
- 用户画像（至少 2-3 个）
- 痛点强度评分
- 付费意愿评分
- 需求验证结论

### 第三步：商业模式设计

**AI 必须主动完成**：

1. **价值主张设计**
   - 定义核心价值（用户能得到什么）
   - 定义情感价值（用户的情感需求）
   - 定义功能价值（用户的功能需求）

2. **盈利模式设计**
   - 设计盈利模式（免费+增值、订阅、按量付费等）
   - 设计免费功能清单
   - 设计付费功能清单
   - 设计定价策略

3. **成本结构分析**
   - 识别主要成本（开发、运营、营销等）
   - 估算成本金额
   - 设计成本控制策略

4. **单位经济分析**
   - 估算用户获取成本（CAC）
   - 估算用户生命周期价值（LTV）
   - 计算 LTV/CAC 比率（目标 > 3:1）

**输出**：
- 价值主张
- 盈利模式设计
- 成本结构分析
- 单位经济分析（LTV/CAC）

### 第四步：竞争优势分析

**AI 必须主动完成**：

1. **技术壁垒**
   - 评估技术门槛
   - 判断是否容易被模仿
   - 识别技术优势

2. **数据壁垒**
   - 评估能否积累有价值的数据
   - 判断数据价值
   - 识别数据优势

3. **网络效应**
   - 评估是否有网络效应
   - 判断网络效应的强度
   - 识别网络效应机会

4. **品牌优势**
   - 评估品牌建设难度
   - 判断品牌价值
   - 识别品牌差异化机会

**输出**：
- 技术壁垒评估
- 数据壁垒评估
- 网络效应评估
- 品牌优势评估

### 第五步：功能优先级评估

**AI 必须主动完成**：

1. **功能价值评估**
   - 评估每个功能的用户价值（1-5 分）
   - 评估每个功能的商业价值（1-5 分）
   - 评估每个功能的开发成本（1-5 分）

2. **优先级排序**
   - P0（MVP - 必须）：用户价值高 + 商业价值高
   - P1（第一版 - 重要）：用户价值高 或 商业价值高
   - P2（后续版本 - 可选）：用户价值中等
   - P3（未来 - 暂不考虑）：用户价值低

3. **MVP 功能清单**
   - 确定 MVP 必须包含的功能
   - 确保 MVP 能够验证核心假设

**输出**：
- 功能价值评估表
- 功能优先级列表
- MVP 功能清单

### 第六步：风险评估

**AI 必须主动完成**：

1. **市场风险**
   - 识别市场风险（竞争、变化等）
   - 评估风险概率和影响
   - 设计应对策略

2. **技术风险**
   - 识别技术风险（成本、依赖等）
   - 评估风险概率和影响
   - 设计应对策略

3. **商业风险**
   - 识别商业风险（获客、转化等）
   - 评估风险概率和影响
   - 设计应对策略

4. **执行风险**
   - 识别执行风险（团队、资源等）
   - 评估风险概率和影响
   - 设计应对策略

**输出**：
- 风险清单
- 风险概率和影响评估
- 应对策略

### 第七步：创建商业价值分析文档

**AI 必须主动完成**：

1. **创建文档**
   - 在项目文档目录创建 `BUSINESS_ANALYSIS.md`
   - 包含所有分析结果
   - 使用清晰的格式和结构

2. **更新项目创建指南**
   - 确保项目创建指南包含商业价值分析阶段
   - 更新检查清单

3. **给出明确建议**
   - 项目是否值得做？（是/否/需要验证）
   - 关键成功因素是什么？
   - 下一步应该做什么？

**输出**：
- `BUSINESS_ANALYSIS.md` 文档
- 明确的建议和决策
- 下一步行动计划

## 重要规则

### 1. 主动分析，不要等待

**错误做法**：
- ❌ "请告诉我目标用户是谁"
- ❌ "你觉得定价多少合适？"
- ❌ "等待你的反馈"

**正确做法**：
- ✅ "基于市场分析，目标用户应该是..."
- ✅ "根据成本分析和竞品定价，建议定价为..."
- ✅ "我已经完成了分析，结论是..."

### 2. 数据驱动，量化分析

**必须包含**：
- 市场规模数据（如果可获取）
- 用户数量估算
- 成本金额估算
- 收入金额估算
- 评分和量化指标

### 3. 多角度思考

**必须考虑**：
- 投资人视角：是否值得投资？
- 用户视角：是否愿意使用和付费？
- 产品视角：如何设计产品？

### 4. 给出明确建议

**必须包含**：
- 项目可行性结论（可行/不可行/需要验证）
- 关键成功因素
- 主要风险
- 下一步行动

## 分析框架参考

### 市场机会分析框架

```
市场规模：大（>10亿）/ 中（1-10亿）/ 小（<1亿）
市场趋势：增长 / 稳定 / 萎缩
竞争格局：激烈 / 中等 / 较少
差异化机会：强 / 中 / 弱
```

### 用户需求分析框架

```
用户群体1：
- 特征：[描述]
- 痛点：[描述]，强度：X/5
- 付费意愿：X/5
- 使用频率：[描述]

用户群体2：
...
```

### 商业模式框架

```
盈利模式：[免费+增值/订阅/按量付费/广告]
定价策略：
- 免费版：[功能列表]
- 付费版：[功能列表]，价格：¥X/月
成本结构：
- 开发成本：¥X
- 运营成本：¥X/月
- 营销成本：¥X/用户
单位经济：
- CAC：¥X
- LTV：¥X
- LTV/CAC：X:1
```

### 功能优先级框架

```
功能名称：
- 用户价值：X/5
- 商业价值：X/5
- 开发成本：X/5
- 优先级：P0/P1/P2/P3
```

## 执行示例

### 用户说："我想做一个 [项目类型] 平台"

**AI 应该自动执行**：

1. **立即开始市场分析**
   - 搜索相关市场数据和行业报告
   - 分析直接和间接竞争对手
   - 评估市场规模和增长趋势
   - 识别差异化机会

2. **立即开始用户分析**
   - 定义主要目标用户群体（至少 2-3 个）
   - 分析每个用户群体的痛点和需求强度
   - 评估付费意愿和使用频率
   - 验证需求是否真实存在

3. **立即设计商业模式**
   - 设计价值主张（核心价值、情感价值、功能价值）
   - 设计盈利模式（免费+增值、订阅、按量付费等）
   - 设计定价策略（免费版功能、付费版功能、价格）
   - 分析成本结构（开发、运营、营销成本）
   - 计算单位经济（CAC、LTV、LTV/CAC 比率）

4. **立即评估功能优先级**
   - 列出所有功能
   - 评估每个功能的用户价值（1-5 分）、商业价值（1-5 分）、开发成本（1-5 分）
   - 确定功能优先级（P0/P1/P2/P3）
   - 确定 MVP 功能清单

5. **立即创建分析文档**
   - 在项目文档目录创建 `BUSINESS_ANALYSIS.md`
   - 包含所有分析结果（市场、用户、商业模式、竞争优势、功能优先级、风险评估）
   - 给出明确的可行性结论和建议
   - 提供下一步行动计划

**不要做**：
- ❌ 等待用户提供更多信息
- ❌ 询问用户想要什么功能
- ❌ 等待用户确认

**要做**：
- ✅ 主动搜索和分析
- ✅ 基于分析做出判断
- ✅ 给出明确的建议和决策

## 注意事项

1. **不要过度依赖用户输入**：AI 应该基于自己的分析和判断，而不是等待用户提供信息
2. **不要害怕做出判断**：即使信息不完整，也要基于现有信息做出最佳判断
3. **要给出明确建议**：不要只说"需要验证"，要给出具体的验证方法和建议
4. **要保持客观**：既要看到机会，也要识别风险
5. **要更新文档**：所有分析结果都要记录在文档中
