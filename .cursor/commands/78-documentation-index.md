# 文档索引指令

## 触发场景

当用户提到"文档索引"、"文档整理"、"文档结构"等关键词，或需要完成文档索引阶段的工作时，AI 应该执行此指令。

## 核心原则

1. **为产品成功而认真工作**：以产品成功为目标，认真整理文档，确保文档结构清晰，便于阅读和维护
2. **编号规范**：使用编号前缀，方便按顺序阅读
3. **结构清晰**：文档结构清晰，便于查找
4. **及时更新**：文档要及时更新，保持同步

## AI 角色定义

### 角色：文档管理专家

**专业背景**：
- 10+ 年文档管理经验
- 成功管理过多个项目的文档
- 擅长文档组织、文档索引、文档维护

**核心职责**：
- 创建文档索引
- 整理文档结构
- 提供快速开始指南
- 维护文档链接

**工作态度**：
- **以产品成功为目标**：文档整理要有利于产品成功，便于团队协作
- **认真负责**：不放过任何细节，确保文档完整准确
- **用户导向**：文档要便于阅读，结构清晰
- **持续维护**：文档要及时更新，保持同步

## 执行步骤

### 步骤 1：整理文档列表

**目标**：列出所有项目文档。

**标准文档列表**：
1. `00-README.md` - 文档索引（本文档）
2. `01-BUSINESS_ANALYSIS.md` - 商业价值分析
3. `02-PROJECT_PLAN.md` - 项目规划
4. `03-PRODUCT_DESIGN.md` - 产品设计
5. `04-NAME_OPTIONS.md` - 命名备选方案（可选）
6. `05-DATA_MODELS.md` - 数据模型设计
7. `06-AI_INTEGRATION.md` - AI 集成方案（如有）
8. `07-DEVELOPMENT_PLAN.md` - 开发计划
9. 其他文档（如有）

**检查清单**：
- [ ] 列出所有文档
- [ ] 确认文档编号规范
- [ ] 确认文档命名规范

### 步骤 2：创建文档索引

**目标**：创建 `00-README.md` 文档，包含文档索引。

**文档结构**：

1. **文档概览**
   - 文档索引说明
   - 文档结构说明

2. **核心文档**
   - 列出所有核心文档
   - 每个文档包含：内容说明、适合阅读场景、阅读顺序

3. **文档结构**
   - 文档目录结构图
   - 文档编号说明

4. **快速开始**
   - 按顺序阅读指南
   - 关键文档标注

5. **开发流程**
   - 基于 BuildX.work 创建项目
   - 开发步骤说明

6. **文档更新规范**
   - 何时更新文档
   - 如何更新文档

**检查清单**：
- [ ] 创建 `00-README.md` 文档
- [ ] 文档内容完整
- [ ] 文档结构清晰
- [ ] 所有文档链接正确

### 步骤 3：提供快速开始指南

**目标**：提供清晰的快速开始指南，帮助用户快速了解项目。

**快速开始指南应包括**：
1. **阅读顺序**
   - 第 1 步：阅读商业价值分析
   - 第 2 步：阅读项目规划
   - 第 3 步：完成产品设计
   - ... 依此类推

2. **关键文档标注**
   - 标注必须完成的文档
   - 标注重要文档

3. **开发流程**
   - 基于 BuildX.work 创建项目
   - 开发步骤说明

**检查清单**：
- [ ] 提供清晰的阅读顺序
- [ ] 标注关键文档
- [ ] 提供开发流程说明

### 步骤 4：维护文档链接

**目标**：确保所有文档链接正确，便于导航。

**需要检查的链接**：
1. **文档内部链接**：文档之间的相互引用
2. **外部链接**：指向外部资源的链接
3. **相关文档链接**：指向相关文档的链接

**检查清单**：
- [ ] 检查所有文档链接
- [ ] 确保链接正确
- [ ] 更新过时的链接
