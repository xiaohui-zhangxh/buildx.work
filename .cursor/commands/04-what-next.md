# What Next - 询问下一步该做什么

## 概述

询问项目下一步该做什么，只回答建议，不执行任何操作。适用于需要了解项目状态、获取下一步建议的场景。

## 🎯 核心原则

1. **只回答，不执行**：只提供下一步建议，不执行任何操作
2. **分析项目状态**：查看项目文档、进度文件、TODO 列表等，了解当前状态
3. **提供明确建议**：给出清晰的下一步行动建议，包括优先级和原因
4. **不修改文件**：不修改任何文件，不执行任何命令（除了读取文件）

## 工作方式

- ✅ **分析状态**：查看项目文档、进度文件、TODO 列表、当前工作状态
- ✅ **提供建议**：给出下一步应该做什么的建议
- ✅ **说明原因**：解释为什么建议这些步骤
- ✅ **优先级排序**：按优先级列出建议
- ❌ **不执行操作**：不执行任何代码修改、文件创建、命令执行等操作
- ❌ **不修改文件**：不修改任何项目文件
- ❌ **不运行测试**：不运行测试或检查

## 触发场景

- 用户询问"下一步该做什么"
- 用户询问"我应该做什么"
- 用户询问"接下来做什么"
- 用户使用指令：`/what-next` 或 `/next-steps`

## 🔍 工作流程

1. **查看当前工作状态**：
   - 读取 `CURRENT_WORK.md` 了解当前正在做什么
   - 查看项目进度文件（如 `docs/phase-*/progress.md`）
   - 查看 TODO 列表（如 `TEST_COVERAGE_TASKS.md`）

2. **分析项目状态**：
   - 查看项目文档了解整体进度
   - 查看功能清单了解已完成和待完成的功能
   - 查看开发计划了解下一步计划

3. **提供建议**：
   - 列出下一步应该做什么（按优先级排序）
   - 说明每个建议的原因和重要性
   - 提供具体的任务描述

4. **不执行任何操作**：
   - 只回答建议，不执行任何操作
   - 不修改文件，不运行命令
   - 等待用户确认后再执行

## 示例

```
用户："/what-next"

AI：
根据当前项目状态，我建议下一步应该：

1. **继续提升测试覆盖率**（高优先级）
   - 原因：当前覆盖率低于目标 85%
   - 建议：完成 TEST_COVERAGE_TASKS.md 中的待办任务
   - 具体任务：为 app/controllers/admin/base_controller.rb 添加测试

2. **完成授权系统开发**（中优先级）
   - 原因：第一阶段认证系统已完成，应该继续第二阶段
   - 建议：查看 docs/phase-2-authorization/progress.md
   - 具体任务：实现角色管理功能

3. **更新项目文档**（低优先级）
   - 原因：开发进度有更新，需要同步文档
   - 建议：更新 CURRENT_WORK.md 和进度文件

请告诉我你想执行哪个任务，或者使用其他指令继续工作。
```

## 与其他指令的区别

- **continue-project-work**：继续完成项目工作，会执行操作
- **smart-go**：智能继续工作，会执行操作
- **autonomous-work**：自主工作，会执行操作
- **what-next**：只回答建议，不执行任何操作

## 注意事项

1. **只回答，不执行**：这是最重要的原则，不要执行任何操作
2. **基于实际状态**：建议应该基于实际的项目状态，不是假设
3. **提供具体建议**：建议应该具体、可操作，不要过于抽象
4. **说明优先级**：按优先级排序，帮助用户决策
5. **等待用户确认**：提供建议后，等待用户确认后再执行
