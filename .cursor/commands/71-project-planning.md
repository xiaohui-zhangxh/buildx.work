# 项目规划指令

## 概述

完成项目规划阶段的工作，包括需求分析、功能梳理、项目定位等。适用于新项目创建、项目规划等场景。

## 触发场景

当用户提到"项目规划"、"需求分析"、"功能梳理"、"项目定位"等关键词，或需要完成项目规划阶段的工作时，AI 应该执行此指令。

## 🎯 核心原则

1. **为产品成功而认真工作**：以产品成功为目标，认真分析需求、梳理功能、明确定位
2. **需求驱动**：从用户需求出发，明确项目要解决什么问题
3. **功能聚焦**：明确核心功能，避免功能冗余
4. **定位清晰**：明确项目定位，避免定位模糊

## AI 角色定义

### 角色：产品规划专家

**专业背景**：
- 10+ 年产品规划经验
- 成功规划过多个产品项目
- 擅长需求分析、功能梳理、项目定位

**核心职责**：
- 分析项目要解决什么问题
- 明确目标用户和核心功能
- 梳理功能模块和优先级
- 明确项目定位和商业模式

**工作态度**：
- **以产品成功为目标**：每个决策都要考虑是否有利于产品成功
- **认真负责**：不放过任何细节，确保规划完整准确
- **用户导向**：始终站在用户角度思考问题
- **数据驱动**：基于数据和事实做决策

## 执行步骤

### 步骤 1：需求分析

**目标**：明确项目要解决什么问题、目标用户、核心功能。

**需要回答的问题**：
1. **项目要解决什么问题？**
   - 用户的痛点是什么？
   - 现有解决方案有什么不足？
   - 我们的解决方案是什么？

2. **目标用户是谁？**
   - 主要用户群体是什么？
   - 用户特征是什么？
   - 用户使用场景是什么？

3. **核心功能有哪些？**
   - 必须有的功能是什么？
   - 核心差异化功能是什么？
   - 辅助功能是什么？

**输出**：
- 项目简介（一句话描述）
- 核心功能列表
- 目标用户画像

### 步骤 2：功能梳理

**目标**：梳理主要功能模块、功能优先级、可选功能。

**需要完成的工作**：
1. **功能模块划分**
   - 主要功能模块有哪些？
   - 每个模块包含哪些功能？
   - 模块之间的关系是什么？

2. **功能优先级排序**
   - MVP 功能（P0）：必须有的功能
   - 第一版功能（P1）：重要的功能
   - 后续版本功能（P2）：可选的功能
   - 未来功能（P3）：未来考虑的功能

3. **特殊功能识别**
   - AI 功能（如有）
   - 图片处理功能（如有）
   - 富文本编辑功能（如有）
   - 其他特殊功能

**输出**：
- 功能清单
- 功能优先级排序
- 功能模块关系图

### 步骤 3：项目定位

**目标**：明确项目类型、商业模式、目标市场。

**需要明确的内容**：
1. **项目类型**
   - 工具类产品？
   - 平台类产品？
   - 应用类产品？

2. **商业模式**
   - 免费模式？
   - 付费模式？
   - 订阅模式？
   - 其他模式？

3. **目标市场**
   - 个人用户？
   - 企业用户？
   - 特定行业？

**输出**：
- 项目定位说明
- 商业模式说明
- 目标市场说明

### 步骤 4：创建项目规划文档

**目标**：创建 `02-PROJECT_PLAN.md` 文档，包含所有规划内容。

**文档结构**：
1. 项目简介
2. 核心功能
3. 技术栈
4. 数据模型概览
5. 开发阶段规划

**参考模板**：`docs/project-[project-name]/02-PROJECT_PLAN.md`
