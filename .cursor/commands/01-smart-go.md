# Smart Go - 智能继续工作

## 概述

继续执行当前任务，自主决定下一步工作，不要停下来汇报进度。适用于需要持续工作、自主决策的场景。

## 🎯 核心原则

1. **继续当前任务**：继续执行正在进行的任务（如提升代码覆盖率、修复问题等）
2. **自主决定下一步**：根据项目状态和最佳实践，自主决定接下来做什么
3. **持续工作**：完成一个任务后立即开始下一个，不要停下来汇报
4. **只在必要时停止**：只有在遇到需要用户决策的问题时才停下来询问

## 工作方式

- ✅ **继续执行**：继续当前正在进行的任务
- ✅ **自主决策**：根据项目状态、TODO 列表、最佳实践，自主决定下一步任务
- ✅ **持续工作**：完成一个任务后立即开始下一个，不要停下来汇报
- ✅ **不要等待**：不要在每个任务完成后等待用户反馈
- ❌ **不要汇报**：不要中途停下来总结进度或询问下一步
- ✅ **只在完成时报告**：只有所有任务完成或遇到需要决策的问题时才报告

## 与 autonomous-work 的区别

- **autonomous-work**：需要先制定完整计划并等待用户确认，然后执行
- **smart-go**：直接继续工作，不需要制定计划或等待确认，自主决定下一步

## 🔍 工作流程

1. **继续当前任务**：继续执行正在进行的任务（如提升代码覆盖率）
2. **检查项目状态**：查看 TODO 列表、项目文档、当前进度
3. **决定下一步**：根据最佳实践和项目状态，自主决定下一个任务
4. **持续执行**：完成任务后立即开始下一个，不中断、不汇报
5. **只在必要时停止**：
   - 所有任务完成时报告
   - 遇到需要用户决策的问题时询问
   - 遇到无法解决的问题时报告

## 示例

```
用户："/smart-go"

AI：
1. 继续执行当前任务（如提升代码覆盖率）
2. 完成任务后，检查 TODO 列表
3. 自主决定下一个任务（如优化邮件模板）
4. 继续执行，不汇报
5. ...持续工作...
6. 所有任务完成或遇到问题时才报告
```

## 注意事项

- 这是一个**执行方式**指令，不是具体任务内容
- 适用于继续当前工作或自主决定下一步工作
- 核心是"继续工作、自主决策、持续执行"
- **关键**：像执行一个连续脚本一样，持续工作，不要中途停下来汇报
