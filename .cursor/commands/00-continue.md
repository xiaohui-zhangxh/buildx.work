# 继续完成项目工作

> 通用指令：继续完成指定项目的下一步工作

## 📋 指令说明

通用指令，用于继续完成指定项目的下一步工作。根据项目文档、进度文件、TODO 列表等，识别下一步应该完成的任务并执行。

### 使用方式

**Cursor 斜线命令**：`/continue-project-work [项目：项目名]`  
**直接描述**：`继续完成项目工作，项目：[项目名]`  
**不指定项目**：`/continue-project-work` 或 `继续完成项目工作`（AI 会列出所有项目让你选择）

## 🎯 核心原则

1. **基于项目状态**：根据项目文档、进度文件、TODO 列表等，识别下一步应该完成的任务
2. **执行具体任务**：不只是建议，而是实际执行任务
3. **更新进度**：完成任务后更新进度文件和文档
4. **保持连续性**：继续之前的工作，不重复已完成的任务

## 🔍 工作流程

### 步骤1：识别项目

1. **如果指定了项目**：使用指定的项目
2. **如果没有指定项目**：
   - 列出所有可用项目
   - 让用户选择项目
   - 或根据当前工作状态自动识别项目

### 步骤2：查看项目状态

1. **查看当前工作状态**：
   - 读取 `CURRENT_WORK.md` 了解当前正在做什么
   - 查看项目进度文件（如 `docs/phase-*/progress.md`）
   - 查看 TODO 列表（如 `TEST_COVERAGE_TASKS.md`）

2. **分析项目状态**：
   - 查看项目文档了解整体进度
   - 查看功能清单了解已完成和待完成的功能
   - 查看开发计划了解下一步计划

### 步骤3：识别下一步任务

根据项目状态，识别下一步应该完成的任务：

1. **优先级排序**：
   - 高优先级：阻塞性任务、关键功能
   - 中优先级：重要功能、优化任务
   - 低优先级：文档更新、代码优化

2. **任务类型**：
   - 功能开发
   - 测试编写
   - Bug 修复
   - 文档更新
   - 代码优化

### 步骤4：执行任务

1. **执行具体任务**：
   - 实现功能
   - 编写测试
   - 修复问题
   - 更新文档

2. **验证结果**：
   - 运行测试确保通过
   - 检查代码质量
   - 验证功能正常

3. **更新进度**：
   - 更新进度文件
   - 更新开发笔记
   - 更新 CURRENT_WORK.md

### 步骤5：报告结果

1. **报告完成情况**：
   - 已完成的任务
   - 遇到的问题和解决方案
   - 下一步建议

2. **询问是否继续**：
   - 询问是否继续下一个任务
   - 或等待用户指示

## 💡 使用示例

```
用户："/continue-project-work"

AI：
1. 查看项目状态（CURRENT_WORK.md、进度文件等）
2. 识别下一步任务（如：提升测试覆盖率）
3. 执行任务（编写测试用例）
4. 验证结果（运行测试）
5. 更新进度（更新进度文件）
6. 报告结果（已完成的任务和下一步建议）
```

## 🔗 与其他指令的区别

- **smart-go**：智能继续工作，自主决定下一步，不汇报进度
- **autonomous-work**：自主工作，需要先制定计划并等待确认
- **what-next**：只回答建议，不执行任何操作
- **continue-project-work**：继续完成项目工作，会执行操作并报告结果

## ⚠️ 注意事项

1. **基于实际状态**：任务应该基于实际的项目状态，不是假设
2. **执行具体任务**：不只是建议，而是实际执行任务
3. **更新进度**：完成任务后及时更新进度文件和文档
4. **保持连续性**：继续之前的工作，不重复已完成的任务
5. **报告结果**：完成任务后报告结果和下一步建议
