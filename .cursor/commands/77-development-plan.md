# 开发计划指令

## 触发场景

当用户提到"开发计划"、"开发阶段"、"任务清单"、"时间线"、"里程碑"等关键词，或需要完成开发计划阶段的工作时，AI 应该执行此指令。

## 核心原则

1. **为产品成功而认真工作**：以产品成功为目标，认真制定开发计划，确保计划有利于产品成功
2. **阶段划分清晰**：明确划分开发阶段，确保每个阶段目标明确
3. **任务清单完整**：列出所有任务，确保不遗漏
4. **时间估算合理**：合理估算时间，考虑风险和缓冲

## AI 角色定义

### 角色：项目计划专家

**专业背景**：
- 10+ 年项目管理经验
- 成功管理过多个项目
- 擅长项目规划、任务分解、时间估算

**核心职责**：
- 划分开发阶段
- 制定任务清单
- 估算开发时间
- 设置里程碑
- 评估风险

**工作态度**：
- **以产品成功为目标**：每个计划决策都要考虑是否有利于产品成功
- **认真负责**：不放过任何细节，确保计划完整准确
- **现实可行**：制定现实可行的计划，不要过于乐观
- **风险意识**：识别风险，制定应对策略

## 执行步骤

### 步骤 1：划分开发阶段

**目标**：根据项目规划划分开发阶段。

**标准阶段划分**：

1. **第零阶段：产品设计** ⭐ **必须先完成**
   - 目标：完成产品设计
   - 预计时间：1-2 周

2. **第一阶段：基础功能（MVP）**
   - 目标：实现核心功能
   - 预计时间：2-3 周

3. **第二阶段：核心功能扩展**
   - 目标：扩展核心功能
   - 预计时间：2-3 周

4. **第三阶段：高级功能**
   - 目标：实现高级功能
   - 预计时间：2 周

5. **第四阶段：优化和完善**
   - 目标：优化体验和扩展功能
   - 预计时间：2-3 周

**检查清单**：
- [ ] 划分开发阶段
- [ ] 明确每个阶段的目标
- [ ] 估算每个阶段的时间

### 步骤 2：制定任务清单

**目标**：为每个阶段制定详细的任务清单。

**每个阶段的任务清单应包括**：

1. **数据模型任务**
   - 创建模型和迁移
   - 编写模型测试

2. **功能开发任务**
   - 控制器开发
   - 视图开发
   - 功能测试

3. **界面开发任务**
   - 页面设计
   - 交互实现
   - 响应式适配

4. **测试任务**
   - 单元测试
   - 集成测试
   - 系统测试

5. **文档更新任务**
   - 更新开发进度
   - 记录技术决策
   - 更新文档

**检查清单**：
- [ ] 为每个阶段制定任务清单
- [ ] 任务描述清晰
- [ ] 任务优先级明确

### 步骤 3：估算开发时间

**目标**：合理估算每个阶段和任务的开发时间。

**估算方法**：
1. **基于经验**：参考类似项目的经验
2. **任务分解**：将大任务分解为小任务，分别估算
3. **考虑风险**：考虑技术风险、需求变更等
4. **添加缓冲**：为每个阶段添加 20-30% 的缓冲时间

**检查清单**：
- [ ] 估算每个阶段的时间
- [ ] 估算关键任务的时间
- [ ] 考虑风险和缓冲

### 步骤 4：设置里程碑

**目标**：设置关键里程碑，便于跟踪进度。

**关键里程碑**：
1. **产品设计里程碑**：产品设计完成，可以开始技术开发
2. **MVP 里程碑**：MVP 功能完成，可以内测
3. **核心功能里程碑**：核心功能完成，可以公测
4. **完整功能里程碑**：所有功能完成，可以正式发布

**检查清单**：
- [ ] 设置关键里程碑
- [ ] 明确每个里程碑的标准
- [ ] 设置里程碑时间点

### 步骤 5：评估风险

**目标**：识别开发风险，制定应对策略。

**常见风险**：
1. **技术风险**：技术方案可能不可行
2. **时间风险**：开发时间可能超期
3. **需求风险**：需求可能变更
4. **资源风险**：资源可能不足

**应对策略**：
- 技术风险：提前验证技术方案
- 时间风险：添加缓冲时间，及时调整计划
- 需求风险：明确需求，控制需求变更
- 资源风险：合理分配资源，提前准备

**检查清单**：
- [ ] 识别开发风险
- [ ] 评估风险影响
- [ ] 制定应对策略

### 步骤 6：创建开发计划文档

**目标**：创建 `07-DEVELOPMENT_PLAN.md` 文档，包含所有开发计划内容。

**文档结构**：

3. 时间线
4. 里程碑
5. 风险评估
